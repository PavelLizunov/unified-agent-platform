# GOAL: Приёмочный тест платформы UAP Hermes (вайб-кодинг)

_Составлено 2026-07-03. Дополняет `runbooks/validation-matrix.md` (инфра-валидация) — здесь
проверяется сквозной пользовательский цикл агента._

**Цель:** доказать, что связка «владелец даёт идею → Hermes кодит на **build-1** → проверяет →
пушит» работает надёжно по всем настроенным областям.

**Принцип оценки:** каждый тест проверяется **объективно** (расположение файла, exit-код, ответ
API, размер контекста) — не по красоте ответа. Агентские промты шлются через
`~/bin/hermes-chat -Q -q "<промт>"` на ops-1; инфра — прямыми проверками (kubectl/ssh/curl).

**Инфра-фикс по ходу D7:** `claude` на build-1 переставал логиниться, когда агент забывал прокинуть
`CLAUDE_CODE_OAUTH_TOKEN` env. Прописал токен в `/etc/environment` на build-1 (pam_env отдаёт его и
non-login ssh, т.е. `build1`-сессиям) → bare `claude -p` авторизован персистентно, надёжность не зависит
от того, помнит ли агент про env.

### Известное ограничение (LLM-потолок, НЕ блокер north-star)
Агент **знает** жёсткое правило (цитирует USER.md дословно), но для **тривиальных** команд
(`hostname`) и **самоописания** якорится на под (`/opt/hermes`) — прозой правила/скилла это не лечится
(усиление USER.md 2026-07-03 не помогло). При этом **реальная работа** (клон, `claude -p`, `git`, `gh`,
сборка) надёжно уходит на build-1 через скилл `remote-build-hosts` — то есть **сам вайб-кодинг-цикл
работает**. Гэп косметический: неверное самоописание + мелкие диагностики в поде.

```bash
# инфра-проверки (из ops-1):
kubectl -n uap-system get pods -l app=hermes-agent
kubectl -n uap-system exec <pod> -c gateway -- /opt/data/.local/bin/build1 "claude --version"
```

## Финальное решение (owner, 2026-07-03): ПРИНЯТО ~98% — вариант A

Владелец принял **~98%** как финальный результат приёмки.

- **D2 (0/2) + D3.1** — осознанно приняты как **косметический LLM-потолок**: агент якорит самоописание
  и тривиальные команды (`hostname`) на под. На вайб-кодинг-цикл (D4–D7, всё на build-1) НЕ влияет.
- **Попытка нативного чистого фикса** (ssh terminal backend, `TERMINAL_ENV=ssh` → build-1, PR #51) НЕ
  активировалась для exec-пути агента (hostname оставался pod-local даже с env+config) и не поддавалась
  независимой валидации через gateway (нет дашборд-логина/Telegram у агента) → откачена (PR #52).
- **Настоящие 100%** потребовали бы переноса Hermes на build-1 (без пода, без дуального контекста);
  владелец сохранил текущую архитектуру (k3s/Flux/SOPS + PVC→R2 бэкапы).

**Приёмка ЗАКРЫТА на ~98%. Вайб-кодинг-платформа принята в эксплуатацию.**
